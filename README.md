# LLM自作入門

## 2章　テキストデータの準備
### 目的
トークナイザーの実装を通じて、入力テキストをトークン化し、さらにトークンIDに変換する方法を学習  
また、トークンIDからトークンを復元する逆変換についても学ぶ

#### トークナイザーの実装
##### V1の実装
vocab をプロパティに持ち、encode, decode メソッドを実装した simple tokenizer を実装  
テキスト encode の際に、vocab に存在しない場合はエラーとなる  
--> 語彙の豊富さを学習により獲得する必要性がわかる  

未知のワードや文章の境界を示すキーワードの導入が必要
- |unk| *unknownからきてる?
- |endoftext| *end of text

##### V2の実装
未知の単語を |unk| トークンで置き換えることができるようにする. 
|unk|, |endoftext| 以外にも特別なトークンは存在する. 
GPTのトークナイザーでは |endoftext| だけ用いる. 
|unk| も使わない ← バイトペアエンコーディング（BPE）を使うため. 

#### バイトペアエンコーディング
テキストをサブワード単位に分割するためのトークナイゼーション手法で、元々はデータ圧縮用のアルゴリズム.   
基本的な仕組みは、
1. 初期化：トークンを文字単位（または最小単位）に分割
"low" -> ['l', 'o', 'w']
2. 反復的なマージ
最も頻繁に隣接して現れるペアを見つけて、1つのトークンにマージする。これを指定回数繰り返す。  
ステップ1: 'l' + 'o' が頻出 -> 'lo' を新トークンとして扱う
'low' -> ['lo', 'w']
ステップ2: 'lo' + 'w' が頻出 -> 'low'を新トークンとして扱う
'low' -> ['low']
3. 語彙の構築
これら一連のプロセスで作成されたトークンが最終的な語彙（vocabulary） になります。
###### BPE の利点
1. 未知のワードに対応可能: 完全に新しい単語に対しても、既知のサブワードの組み合わせで表現できる.  
2. 語彙サイズの制御: マージ回数で語彙サイズを調整可能
3. 頻出語は1トークンになるので、効率的に扱える

### 用語集
トークン：
    LLMに認識させるテキストの単位。rawデータからトークナイザーが適した形でパース
    トークンはその後、トークンIDという整数値に置き換えられる
トークンID：
    個々のトークンを一意の整数値に変換したもの。
